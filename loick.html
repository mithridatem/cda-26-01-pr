AI Website Cloner Template

Un template open-source permettant de cloner n'importe quel site web en une seule commande, grâce à des agents IA de programmation. Le résultat est une base de code Next.js propre et moderne.

Thème : Intelligence Artificielle / Outils développeur / Automatisation / Next.js

Date : Juin 2026 (dernière release : v0.3.1 – 30 mars 2026)

Auteur / Source : JCodesMore – GitHub (18 400 ⭐ · 2 800 forks)

Description :

Ce dépôt est un template GitHub (utilisable via le bouton Use this template) qui fournit un pipeline complet pour reconstruire n'importe quel site web dans un projet Next.js 16 + TypeScript + Tailwind CSS v4. L'agent IA recommandé est Claude Code avec le modèle Opus 4.7, mais le template est compatible avec une douzaine d'autres agents : Codex CLI, Cursor, Windsurf, Gemini CLI, GitHub Copilot, etc.

Le processus se déclenche avec la commande /clone-website <url> et s'exécute en plusieurs phases automatisées : reconnaissance visuelle et extraction des design tokens, téléchargement des assets, rédaction de specs de composants avec les valeurs CSS calculées exactes, construction parallèle via des agents en git worktrees, puis assemblage final avec comparaison visuelle par rapport au site original.

Les cas d'usage principaux sont la migration de plateforme (WordPress, Webflow, Squarespace → Next.js), la récupération de code source perdu, et l'apprentissage par déconstruction de sites de production. Le projet précise explicitement ce pour quoi il n'est pas fait : hameçonnage, usurpation d'identité, ou violation des conditions d'utilisation des sites cibles.

Analyse :

Ce projet illustre concrètement la montée en puissance des agents IA de programmation pour automatiser des tâches complexes de reverse-engineering front-end. La popularité exceptionnelle du dépôt (18k+ étoiles en quelques mois) confirme un fort intérêt de la communauté. Son architecture multi-agents avec des worktrees git parallèles est une approche technique avancée et réutilisable dans d'autres contextes d'automatisation.

Sur le plan éthique et légal, le projet soulève des questions importantes : la reproduction de sites tiers peut enfreindre des droits d'auteur ou des CGU. Il est indispensable de vérifier la légitimité de l'usage avant toute utilisation (sites personnels, migrations autorisées, projets pédagogiques). À surveiller pour les équipes souhaitant moderniser rapidement un parc applicatif vieillissant.
